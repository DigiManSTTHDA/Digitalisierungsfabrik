**Status:** Complete

## 1. Epic Summary

### What was implemented

Epic 08 adds the second of four cognitive phases to the Digitalisierungsfabrik: the **Structuring phase**. After the Exploration phase captures free-text process knowledge through a guided interview, the Structuring phase transforms that knowledge into a formal process definition — similar to a BPMN process diagram, but text-based.

The system now conducts a structured dialogue with the user to decompose a business process into individual steps, identify decision points (e.g., "Is the expense above 500?"), loops, and exception paths. Each element is stored as a formal data structure that downstream phases can refine.

Additionally, Epic 08 adds **project deletion** — both individual and bulk — so users can clean up their project list.

### Why this matters

The Structuring phase is the critical bridge between "understanding the process" (Exploration) and "specifying how to automate it" (Specification). Without it, the system cannot produce the structured input that EMMA RPA needs. This epic moves the prototype from interview-only to actual process modelling.

## 2. Implemented Components

### StructuringMode (LLM-based Cognitive Mode)

**Purpose:** Guides the user through process decomposition via AI-driven dialogue. The LLM reads the completed Exploration Artifact and systematically builds a Structure Artifact containing process steps, decisions, loops, and exceptions.

**Key files:** `backend/modes/structuring.py`, `backend/prompts/structuring.md`

**Interaction:** The Orchestrator activates StructuringMode after the Moderator confirms the phase transition from Exploration. The mode reads the Exploration Artifact as read-only context, generates RFC 6902 JSON Patch operations to build the Structure Artifact, and signals phase completion when all steps are user-validated.

### Structure Artifact Schema Enhancement

**Purpose:** The Structure Artifact data model was extended with a `prozesszusammenfassung` (process summary) field, as required by the SDD. This is a human-readable free-text summary of the entire process, generated by the LLM at the end of the Structuring phase.

**Key files:** `backend/artifacts/models.py`, `backend/artifacts/template_schema.py`

### Structure Artifact Rendering (Frontend)

**Purpose:** The artifact pane now displays Structure Artifact content in a structured, readable format — each process step shows its type (action/decision/loop/exception) as a colour-coded badge, its position in the sequence, successor connections, and any documented risks.

**Key files:** `frontend/src/components/ArtifactTab.tsx`

### Project Deletion (Backend + Frontend)

**Purpose:** Users can now delete individual projects or select multiple projects for bulk deletion. A confirmation dialog prevents accidental data loss.

**Key files:** `backend/persistence/project_repository.py`, `backend/api/router.py`, `frontend/src/App.tsx`, `frontend/src/components/ConfirmDialog.tsx`

## 12. Next Steps

**Epic 09: Specification Mode** will implement the third cognitive phase. It transforms the Structure Artifact's process steps into technically precise EMMA RPA specifications — each step gets concrete actions, parameters, expected outcomes, timeouts, and error handling.

This is the most technically detailed phase: the output must be precise enough for an RPA tool to execute without human interpretation. Epic 09 bridges the gap between "what the process looks like" (Structure Artifact) and "exactly how a robot should execute it" (Algorithm Artifact).

After Epic 09, the system will have three of four phases working. Epic 10 (Validation) will add consistency checking, and Epic 11 will stabilize the complete end-to-end flow.
