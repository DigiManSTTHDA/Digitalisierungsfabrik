# Implementation Timeline

This timeline defines the epic sequence for the working prototype. Each epic must end with a visible, user-testable increment.

## Timeline

| Epic | Title | Increment visible to user |
|---|---|---|
| `00` | Bootstrap Shell | App starts locally and shows a working frontend-backend shell |
| `01` | Project Persistence | Projects can be created, reloaded, and restored from storage |
| `02` | Artifact Engine | Artifact changes, versions, and restore behavior are inspectable |
| `03` | Conversation Skeleton | Users can send messages and see live deterministic system updates |
| `04` | Exploration Workflow | Users can build the exploration artifact through conversation and file input |
| `05` | Structuring Workflow | Users can move into structure generation and inspect a process model |
| `06` | Specification Workflow | Users can inspect an EMMA-aligned algorithm artifact |
| `07` | Validation And Closure | Users can validate, iterate, and finish a project with exportable results |
| `08` | Prototype Hardening | Users can run the full prototype reliably with recovery and observability support |

## Detail
- Detailed epic definitions live in `agent-docs/epics/`.
- Story breakdown is intentionally deferred until epic planning starts.
